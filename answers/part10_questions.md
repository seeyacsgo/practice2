## Вопрос 1

Что выведет этот код? Объясните порядок выполнения блоков.

```java
class Demo {
    static { System.out.print("A "); }
    { System.out.print("B "); }
    Demo() { System.out.print("C "); }
    public static void main(String[] args) {
        System.out.print("D ");
        new Demo();
        System.out.print("E ");
        new Demo();
    }
}
```

**Предсказанный вывод:**

```
A D B C E B C 
```

**Объяснение:**

static {} — выполняется один раз при загрузке класса → A

main() → D

new Demo(): сначала блок экземпляра {} → B, затем конструктор → C

E

new Demo(): снова B C

## Вопрос 2

Класс `Child` в пакете `b`, класс `Parent` в пакете `a`. `Child extends Parent`. Какие члены `Parent` доступны в `Child`? А какие доступны в другом классе пакета `b`, который НЕ наследует `Parent`?

**Ваш ответ:**

В Child (наследует Parent): доступны public, protected, package-private (если в одном пакете  нет, они в разных).

В другом классе пакета b (не наследник): доступны только public.

## Вопрос 4

Что выведет этот код? Объясните, что такое String Pool и как работает `intern()`.

```java
String a = "Hello";
String b = new String("Hello");
String c = b.intern();
System.out.println(a == b);
System.out.println(a == c);
System.out.println(b == c);
```

**Предсказанный вывод:**

```
false
true
false
```

**Объяснение:**

String Pool — область в heap, где хранятся уникальные строковые литералы.

a — литерал, попадает в пул.

b — new String() создаёт новый объект в куче (не в пуле).

c = b.intern() — возвращает ссылку на объект из пула (тот же, что и a).

a == b — разные объекты → false

a == c — оба из пула → true

b == c — куча vs пул → false

## Вопрос 6

В чём разница между `StringBuilder` и `StringBuffer`? Приведите конкретный сценарий, когда нужен именно `StringBuffer`.

**Ваш ответ:**

    StringBuilder — несинхронизированный, быстрее.

    StringBuffer — синхронизированный (thread-safe), медленнее.

Сценарий для StringBuffer: многопоточное приложение, где одна строка изменяется из нескольких потоков.

## Вопрос 7

Что выведет этот код? Объясните, что происходит с переменной `x` в лямбде.

```java
int x = 10;
Runnable r = () -> System.out.println(x);
// x = 20;  // что будет, если раскомментировать?
r.run();
```

**Предсказанный вывод:**

```
10
```

**Объяснение:**

Лямбда захватывает переменную x  она должна быть effectively final (не изменяться после инициализации). Если раскомментировать x = 20, код не скомпилируется с ошибкой: "Variable used in lambda should be effectively final".

## Вопрос 8

Напишите пример лямбда-выражения, которое нельзя заменить на ссылку на метод. Объясните почему.

**Ваш пример и объяснение:**

Function<Integer, Integer> f = x -> x * x + 1;

Нельзя заменить на ссылку на метод, так как лямбда выполняет составную операцию (умножение и сложение), а не просто вызов одного существующего метода.

## Вопрос 9

Чем `EnumMap` быстрее `HashMap` для ключей-enum? Опишите внутреннюю реализацию в одном предложении.

**Ваш ответ:**

EnumMap внутри использует обычный массив, индексированный по ordinal() enum-константы, что даёт доступ за O(1) без вычисления хэш-кода.

## Вопрос 10

Дан `record Point(int x, int y) {}`. Что сгенерирует компилятор автоматически? Можно ли добавить в record изменяемое поле? Можно ли наследоваться от record?

**Ваш ответ:**

Генерирует автоматически: конструктор с параметрами, геттеры x(), y(), equals(), hashCode(), toString().

Можно ли добавить изменяемое поле? Нет — record предназначен для неизменяемых данных.

Можно ли наследоваться от record? Нет — record неявно final.
