# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор | Обоснование |
|----------|-----------|-------------|
| Все животные имеют имя и возраст, но звучат по-разному | **Абстрактный класс** | Общее состояние (поля name, age) + отношение «является» животным |
| Некоторые объекты можно сериализовать | **Интерфейс** | Отношение «умеет» + множественная реализация |
| Все транспортные средства имеют скорость и топливо, но двигаются по-разному | **Абстрактный класс** | Общие поля + отношение «является» ТС |
| Робот, посудомоечная машина, телефон — все умеют подключаться к Wi-Fi | **Интерфейс** | Отношение «умеет» + разные иерархии классов |
| Все фигуры с площадью и периметром, у всех есть цвет | **Абстрактный класс** | Общее состояние (цвет) + отношение «является» фигурой |
| Класс должен уметь летать, плавать и ездить | **Интерфейсы** | Множественное наследование поведения (can-do) |
